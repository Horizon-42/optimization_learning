Structured linear and discrete models

Linear and combinatorial optimization

Some problems are not best treated as smooth nonlinear objectives. Linear programs, assignment problems, and mixed-integer models expose algebraic structure that specialized solvers can exploit directly.

Overview Constrained Global optimization Least squares Univariate Root finding Structured Advanced

Core Idea

Structured optimization makes constraints and decision-variable types explicit. For a linear program, SciPy's tutorial uses the standard form

\[ \begin{aligned} \min_x\quad & c^\mathsf{T}x\\ \text{subject to}\quad & A_{ub}x\leq b_{ub},\\ & A_{eq}x=b_{eq},\\ & \ell\leq x\leq u. \end{aligned} \]

Assignment and mixed-integer models add discrete choices. They are still algebraic, but the feasible set is no longer a smooth surface.

Linear Programming

linprog minimizes a linear objective over linear equality, inequality, and bound constraints. In two dimensions, the feasible set is a polygon; in higher dimensions it is a polytope.

Assignment Problems

The linear sum assignment problem chooses one entry from each row without reusing columns:

\[ \min_X \sum_i\sum_j C_{ij}X_{ij}, \qquad X_{ij}\in\{0,1\}. \]

Mixed-Integer Linear Programming

Mixed-integer linear programming keeps some decision variables integer. The tutorial's knapsack example uses binary variables:

\[ \max \sum_i v_i x_i \quad \text{subject to} \quad \sum_i s_i x_i\leq C,\qquad x_i\in\{0,1\}. \]

Using SciPy

The APIs are different because the problem structures are different: matrix form for linear programming, a cost matrix for assignment, and integrality flags for MILP. 

import numpy as np
from scipy.optimize import Bounds, LinearConstraint, linear_sum_assignment, linprog, milp

# Linear programming: maximize 3*x0 + 2*x1 by minimizing its negative.
c = np.array([-3.0, -2.0])
A_ub = np.array([[1.0, 1.0], [2.0, 1.0], [1.0, 0.0], [0.0, 1.0]])
b_ub = np.array([4.0, 5.0, 3.0, 3.0])
lp = linprog(c, A_ub=A_ub, b_ub=b_ub, bounds=[(0.0, None), (0.0, None)])

# Assignment: rows are swimming styles, columns are students.
cost = np.array([[43.5, 45.5, 43.4, 46.5, 46.3],
                 [47.1, 42.1, 39.1, 44.1, 47.8],
                 [48.4, 49.6, 42.1, 44.5, 50.4],
                 [38.2, 36.8, 43.2, 41.2, 37.2]])
row_ind, col_ind = linear_sum_assignment(cost)

# MILP knapsack: maximize value under capacity, with binary variables.
sizes = np.array([21, 11, 15, 9, 34, 25, 41, 52])
values = np.array([22, 12, 16, 10, 35, 26, 42, 53])
capacity = 100
milp_result = milp(
    c=-values,
    constraints=LinearConstraint(sizes, lb=0, ub=capacity),
    integrality=np.ones_like(values),
    bounds=Bounds(0, 1),
)

When To Use It

Tool Use when Modeling habit
linprog Objective and constraints are linear and variables are continuous. Convert maximization to minimization by negating the objective.
linear_sum_assignment You need one-to-one matching with minimum total cost. Put the assignable groups on rows and columns of a cost matrix.
milp Some variables must be integer or binary. Use bounds plus integrality to encode discrete decisions.
Practical habit: model first, solve second. Most structured optimization bugs are sign errors, wrong inequality direction, or forgotten integrality constraints.
