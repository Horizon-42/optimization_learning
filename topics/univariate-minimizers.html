Scalar search with brackets and bounds

Univariate minimization tutorial

When the input is one scalar, the optimizer can reason directly about an interval on a line. This tutorial builds from the idea of bracketing a local minimum, then compares golden-section search, Brent search, and bounded scalar minimization in SciPy.

Start With A Line

A univariate minimizer solves a problem where the input \(x\) is a single number. The objective is still a scalar function, but the search space is now a line instead of a plane or high-dimensional parameter vector:

\[ \min_{x\in\mathbb{R}} f(x). \]

This is the same shape of problem that appears inside line search. In BFGS or Newton-CG, the multivariate objective is sliced along a direction to form \(\phi(\alpha)\). Here there is no hidden vector behind the notation: the original problem is already one-dimensional.

That makes plotting and bracketing unusually valuable. If evaluating \(f(x)\) is cheap, a quick plot often tells you whether the interval has one valley, several valleys, or no visible minimum at all.

What A Bracket Means

The tempting shortcut is to give the solver two endpoints and expect it to minimize between them. That is not what an unconstrained bracket means in SciPy. For brent and golden, a strong bracket is three points:

\[ a\lt b\lt c, \qquad f(a)\gt f(b)\lt f(c). \]

The middle point \(b\) is lower than both sides, so the three values give evidence that a local minimum is nearby. A two-point bracket can be used as a starting hint for a downhill bracket search, but it is not a hard interval constraint. The reported minimizer is not guaranteed to stay between the two initial points.

Distinction worth keeping: a bracket helps the solver locate a local valley. Bounds tell the solver where it is allowed to evaluate.

Golden-Section Search

Golden-section search is the cleanest interval-shrinking idea. Once a local minimum is bracketed, the method evaluates interior points and discards the side that cannot contain the bracketed minimum.

The special scalar is

\[ \tau=\frac{\sqrt{5}-1}{2}\approx 0.618. \]

This value is chosen so that after the interval shrinks, one old interior point can be reused as a new interior point. That reuse is the whole trick: every shrink needs only one new function evaluation instead of two.

Golden-section search is dependable and easy to reason about, but it does not try to learn the curve's shape. It only shrinks intervals.

Brent Adds A Faster Guess

Brent's method keeps the bracketed-search safety idea, then adds a faster move when the recent function values look like they can be fit by a parabola. Three nearby points determine a quadratic model. If that parabola predicts a useful point inside the current search region, Brent tries it.

The failed shortcut would be to always trust the parabola. A quadratic fit can be misleading if the sampled points are awkward or the function is not locally quadratic enough. Brent therefore falls back to golden-section-style progress when interpolation is not trustworthy.

Count Function Evaluations

In scalar black-box optimization, the most practical cost is often not algebra. It is how many times the objective must be called. A useful diagnostic is the best objective gap after \(k\) probes:

\[ \begin{aligned} f_{\mathrm{best},k} &= \min_{1\leq i\leq k} f(x_i),\\ \Delta_k &= f_{\mathrm{best},k}-f^\star. \end{aligned} \]

The gap \(\Delta_k\) asks a simple question: after spending \(k\) function evaluations, how far is the best value we have seen from the known optimum in this example?

Bounds Are Different From Brackets

Sometimes the feasible interval is part of the problem statement:

\[ \min_{a\leq x\leq b} f(x). \]

In that case, use method="bounded" and pass finite bounds=(a, b). Unlike a bracket, these bounds are a hard search interval. The method only looks for a local minimum inside that interval.

The SciPy tutorial demonstrates this by minimizing the Bessel function \(J_1(x)\) over \([4,7]\):

\[ \min_{4\leq x\leq 7} J_1(x). \]

Use Minimize Scalar In SciPy

The entry point is scipy.optimize.minimize_scalar. If you pass bounds without a method, SciPy defaults to the bounded method. Otherwise it defaults to Brent. The explicit version below names each method so the behavior is visible: 

from scipy import optimize as opt
from scipy import special

def f(x):
    return (x - 2) * (x + 1)**2

bracket = (-1.5, 0.0, 2.2)

brent = opt.minimize_scalar(
    f,
    method="brent",
    bracket=bracket,
)

golden = opt.minimize_scalar(
    f,
    method="golden",
    bracket=bracket,
)

bounded = opt.minimize_scalar(
    special.j1,
    method="bounded",
    bounds=(4.0, 7.0),
)

print(brent.x, brent.fun)
print(golden.x, golden.fun)
print(bounded.x, bounded.fun)

Read the result as a local scalar minimization result. result.x is the reported scalar minimizer, result.fun is the objective value there, and result.nfev counts objective evaluations.

When To Use It

Method Use when Watch for
brent You have a one-dimensional unconstrained objective and can provide a meaningful bracket. A bracket is local evidence, not a guarantee of the global minimum of a multimodal function.
golden You want a simple bracket-shrinking method or an educational baseline. It usually needs more function evaluations than Brent on smooth problems.
bounded The answer must lie inside a fixed finite interval \([a,b]\). It only searches inside the interval, so a bad interval can hide the better valley.
Practical habit: plot the scalar objective first when it is cheap. One-dimensional plots often reveal whether a bracket is sensible, whether the interval contains several valleys, or whether the problem really needs a global search.
