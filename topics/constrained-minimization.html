Local minimization with constraints

Constrained minimization tutorial

Constrained minimization keeps the local-search goal of reducing a scalar objective, but every step must also respect bounds, equalities, and inequalities. This tutorial explains the feasible-set idea first, then shows how trust-constr and SLSQP encode the same constrained Rosenbrock problem in SciPy.

From Unconstrained To Feasible

In an unconstrained problem, a step is judged mostly by whether it lowers the objective \(f(x)\). In a constrained problem, lowering the objective is not enough. The point must also be feasible, meaning it satisfies the rules of the problem.

Start with a current point \(x\). The objective \(f(x)\) is still the scalar value you want to make small. Bounds restrict each variable directly. Constraint functions \(c(x)\) describe extra quantities that must stay between lower and upper limits:

\[ \begin{aligned} \min_x\quad & f(x),\\ c^\ell &\leq c(x)\leq c^u,\\ x^\ell &\leq x\leq x^u. \end{aligned} \]

The new object is the feasible set: all \(x\) values that satisfy those bounds and constraints. A good constrained step must improve the objective while staying in, or moving back toward, that feasible set.

Mental model: unconstrained minimization asks "which point is low?" Constrained minimization asks "which feasible point is low?"

How To Read Constraints

Bounds, inequalities, and equalities look similar in code, but they play different roles.

A bound is a direct limit on one coordinate:

\[ 0\leq x_0\leq 1. \]

An inequality leaves room. For example, \(x_0+2x_1\leq 1\) allows every point on one side of a line:

\[ x_0+2x_1\leq 1. \]

An equality removes more freedom. The condition \(2x_0+x_1=1\) forces the solution to lie exactly on a line:

\[ 2x_0+x_1=1. \]

The tempting shortcut is to ignore the constraints, minimize the objective, and then project the answer back into the allowed region. That can fail because the best feasible point is usually not the projection of the unconstrained minimum. The constraint geometry and the objective geometry have to be considered together.

The Tutorial Problem

The SciPy constrained examples use the two-variable Rosenbrock objective. By itself, Rosenbrock wants to reach the valley minimum near \((1,1)\). The constraints below change the question: among the points that obey the line, curve, and bound rules, which one has the lowest Rosenbrock value?

\[ \begin{aligned} \min_{x_0,x_1}\quad &100(x_1-x_0^2)^2+(1-x_0)^2,\\ x_0+2x_1&\leq1,\\ x_0^2+x_1&\leq1,\\ x_0^2-x_1&\leq1,\\ 2x_0+x_1&=1,\\ 0\leq x_0&\leq1,\\ -0.5\leq x_1&\leq2. \end{aligned} \]

Meaning: the optimizer is not free to chase the lowest contour anywhere in the plane. It must search along the feasible geometry carved out by the constraints.

trust-constr And SLSQP

SciPy offers several constrained local optimizers. The tutorial contrast here is between trust-constr and SLSQP.

trust-constr is the more expressive modern interface. It uses Bounds, LinearConstraint, and NonlinearConstraint objects. That object-style interface lets you name the lower and upper limits for each constraint and provide derivatives when the problem needs them.

SLSQP is an SQP-style method with a compact dictionary interface. It repeatedly solves quadratic approximations to the constrained problem. For smaller smooth problems, the dictionary style is often convenient.

Method Constraint interface Good fit
trust-constr Bounds, LinearConstraint, NonlinearConstraint General smooth constrained problems, especially when Hessians, Hessian-vector products, or sparsity are useful.
SLSQP Dictionaries with type, fun, and optional jac Smaller smooth constrained problems where a mature SQP-style method is convenient.

Constraint Slack

Inequality constraints are often easiest to audit through slack. Suppose a constraint says \(c_j(x)\leq c_j^u\). The slack is the unused room before the upper limit:

\[ s_j(x)=c_j^u-c_j(x). \]

Now the sign has a clear meaning:

\[ \begin{aligned} s_j(x)>0 &\quad \text{constraint has room},\\ s_j(x)=0 &\quad \text{constraint is active},\\ s_j(x)<0 &\quad \text{constraint is violated}. \end{aligned} \]

This is a useful failed-shortcut check. If the objective value looks good but one slack is negative, the optimizer has not solved the constrained problem yet. It has found a low infeasible point.

Use Constrained Minimization In SciPy

The same mathematical problem can be represented with modern constraint objects for trust-constr, or with dictionary constraints for SLSQP. 

import numpy as np
from scipy.optimize import Bounds, LinearConstraint, NonlinearConstraint, minimize

def rosen(x):
    return 100.0 * (x[1] - x[0]**2)**2 + (1.0 - x[0])**2

def rosen_der(x):
    return np.array([
        -400.0 * x[0] * (x[1] - x[0]**2) - 2.0 * (1.0 - x[0]),
        200.0 * (x[1] - x[0]**2),
    ])

bounds = Bounds([0.0, -0.5], [1.0, 2.0])

linear = LinearConstraint(
    [[1.0, 2.0], [2.0, 1.0]],
    [-np.inf, 1.0],
    [1.0, 1.0],
)

nonlinear = NonlinearConstraint(
    lambda x: np.array([x[0]**2 + x[1], x[0]**2 - x[1]]),
    [-np.inf, -np.inf],
    [1.0, 1.0],
)

x0 = np.array([0.5, 0.0])

trust = minimize(
    rosen,
    x0=x0,
    method="trust-constr",
    jac=rosen_der,
    constraints=[linear, nonlinear],
    bounds=bounds,
)

ineq_cons = {
    "type": "ineq",
    "fun": lambda x: np.array([
        1.0 - x[0] - 2.0 * x[1],
        1.0 - x[0]**2 - x[1],
        1.0 - x[0]**2 + x[1],
    ]),
}

eq_cons = {
    "type": "eq",
    "fun": lambda x: np.array([2.0 * x[0] + x[1] - 1.0]),
}

slsqp = minimize(
    rosen,
    x0=x0,
    method="SLSQP",
    jac=rosen_der,
    constraints=[eq_cons, ineq_cons],
    bounds=[(0.0, 1.0), (-0.5, 2.0)],
)

print(trust.x, trust.fun)
print(slsqp.x, slsqp.fun)

The sign convention differs by interface. In LinearConstraint and NonlinearConstraint, you give lower and upper bounds directly. In an SLSQP inequality dictionary, the function should be nonnegative when the constraint is satisfied.

When To Use It

Use constrained minimization when Prefer another family when
The objective is smooth and the feasible set is described by bounds, equalities, or inequalities. The objective is naturally a residual vector; then least_squares may expose better structure.
You can provide analytic gradients or Jacobians for the objective and constraints. The feasible set is combinatorial or discrete; then linear, assignment, or mixed-integer tools may fit better.
Feasibility matters as much as objective reduction. The search space has many local minima and broad global exploration is the first concern.
Practical habit: inspect both result.fun and constraint violation. In constrained optimization, a pretty objective value is only meaningful when the point is feasible.
