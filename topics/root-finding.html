Zeros, fixed points, and nonlinear systems

Root finding

Root finding solves equations directly. Instead of minimizing a scalar score, the goal is to make a scalar function or residual vector equal to zero.

Overview Constrained Global optimization Least squares Univariate Root finding Structured Advanced

Core Idea

For a scalar function, root finding solves

\[ f(x^\star)=0. \]

For systems, the unknown is a vector and every component of the residual must vanish:

\[ F(x^\star)=0,\qquad F:\mathbb{R}^n\to\mathbb{R}^n. \]

This is related to optimization, but not identical. Minimizing \(\|F(x)\|^2\) can be useful, yet root solvers exploit equation structure more directly.

Scalar Functions

Bracketed scalar methods use a sign change \(f(a)f(b)<0\), which gives strong safety guarantees for continuous functions. Newton-style methods use derivatives and can move faster when the initial guess is good:

\[ x_{k+1}=x_k-\frac{f(x_k)}{f'(x_k)}. \]

Sets of Equations

The tutorial's system example solves two equations in two unknowns:

\[ \begin{aligned} x_0\cos(x_1)&=4,\\ x_0x_1-x_1&=5. \end{aligned} \]

Geometrically, the root is where the zero-contours of the residual components intersect.

Fixed Points

A fixed point solves \(g(x)=x\). SciPy's fixed_point works directly with this form, while a root solver can handle the equivalent residual:

\[ g(x)-x=0. \]

In practice, fixed-point iteration is attractive when the problem is naturally written as an update rule, and a root solver is attractive when the residual and Jacobian are the clearer objects.

Large Nonlinear Systems

For high-dimensional residuals, forming and factoring a dense Jacobian is often the wrong cost model. Krylov root methods use matrix-vector products and can accept a preconditioner \(M\) that approximately applies \(J^{-1}\):

\[ M \approx J(x)^{-1}. \]

Using SciPy

Use scalar root routines when the unknown is one-dimensional, and root when the residual is vector-valued. 

import numpy as np
from scipy.optimize import root, root_scalar

def f(x):
    return x + 2.0 * np.cos(x)

def fp(x):
    return 1.0 - 2.0 * np.sin(x)

safe = root_scalar(f, bracket=(-2.0, 0.0), method="brentq")
fast = root_scalar(f, x0=0.25, fprime=fp, method="newton")

def F(x):
    return np.array([x[0] * np.cos(x[1]) - 4.0, x[0] * x[1] - x[1] - 5.0])

def J(x):
    return np.array([[np.cos(x[1]), -x[0] * np.sin(x[1])], [x[1], x[0] - 1.0]])

system = root(F, x0=np.array([1.0, 1.0]), jac=J)

When To Use It

Problem shape Useful SciPy tools Key habit
Scalar equation with a sign change root_scalar, brentq, bisect Prefer a bracket when reliability matters.
Scalar equation with derivatives newton, root_scalar(method="newton") Check sensitivity to the initial guess.
Vector residual system root Scale residual components and provide a Jacobian when possible.
Large sparse or matrix-free system root(method="krylov") Invest in preconditioning before blaming the nonlinear solver.
Practical habit: inspect \(\|F(x)\|\), not just the success flag. Root finding is done when the residual is small at a point you trust.
