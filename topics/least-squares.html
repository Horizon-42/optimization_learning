Residual fitting with bounds and robust loss

Least-squares tutorial

Least squares is the optimizer to reach for when the objective is really a collection of residual errors. This tutorial starts from ordinary model fitting, then shows why SciPy's least_squares can use residual structure, Jacobians, bounds, and robust losses.

From One Loss To Many Residuals

Suppose a model predicts observations. For input \(u_i\), the model prediction is \(m(x,u_i)\), where \(x\) contains the parameters we want to fit. The observed value is \(y_i\). A residual is the prediction error for one observation:

\[ r_i(x)=m(x,u_i)-y_i. \]

If there are \(m\) observations, the residual function returns a vector, not one scalar:

\[ r(x)= \begin{bmatrix} r_1(x)\\ r_2(x)\\ \vdots\\ r_m(x) \end{bmatrix}. \]

This is the key distinction from minimize. With minimize, you hand SciPy a scalar objective. With least_squares, you hand SciPy the residual vector and let the solver exploit its special structure.

Mental model: least_squares is not asking "what is the cost?" It is asking "what are all the individual errors that make up the cost?"

Why Square The Residuals?

A tempting shortcut is to add the residuals directly. That fails because positive and negative errors can cancel. A model with one large positive error and one large negative error might look perfect after summation.

Squaring avoids cancellation and makes the objective smooth near zero. With the ordinary linear loss, least squares minimizes

\[ \begin{aligned} F(x) &= \frac{1}{2}\sum_{i=1}^{m} r_i(x)^2\\ &= \frac{1}{2}\|r(x)\|_2^2. \end{aligned} \]

The factor \(1/2\) is not decoration. It cancels the \(2\) produced when differentiating a square. If \(J\) is the residual Jacobian, with entries \(J_{ij}=\partial r_i/\partial x_j\), then

\[ \begin{aligned} \frac{\partial F}{\partial x_j} &= \sum_{i=1}^{m} r_i(x)\frac{\partial r_i}{\partial x_j},\\ \nabla F(x) &= J(x)^\mathsf{T}r(x). \end{aligned} \]

Meaning: the gradient of the scalar cost is built from the residuals and the Jacobian. This is why providing residual structure gives the solver more information than hiding everything inside one scalar function.

A Bounded Enzyme-Reaction Fit

The SciPy tutorial uses an enzymatic reaction model with eleven measurements. The input is \(u_i\), the observed response is \(y_i\), and the four fitted parameters are \(x=(x_0,x_1,x_2,x_3)^\mathsf{T}\). The model prediction is a ratio:

\[ m(x,u_i) = \frac{x_0(u_i^2+x_1u_i)} {u_i^2+x_2u_i+x_3}. \]

The residual for each data point is model minus measurement:

\[ r_i(x)=m(x,u_i)-y_i. \]

The bounds \(0\leq x_j\leq 100\) encode a simple physical rule: these parameters should not become negative or enormous while the solver is trying to match the observations.

The Jacobian Turns Errors Into A Step

At the current parameter vector \(x\), the solver wants a step \(p\). The exact residual vector \(r(x+p)\) may be nonlinear, so the Gauss-Newton idea is to replace it with a first-order model:

\[ r(x+p)\approx r(x)+J(x)p. \]

Now the hard nonlinear fitting problem has a local linearized least-squares subproblem:

\[ \min_p \frac{1}{2}\|r(x)+J(x)p\|_2^2. \]

If we ignore bounds for a moment and differentiate that subproblem with respect to \(p\), the normal equations appear:

\[ J^\mathsf{T}Jp=-J^\mathsf{T}r. \]

This looks like Newton's method, but the matrix \(J^\mathsf{T}J\) comes from residual sensitivities rather than from forming the full Hessian of the scalar cost. That approximation is the workhorse of nonlinear least squares.

Bounds And Solver Choices

SciPy's least_squares has three algorithm families. The default method="trf" is the safest general choice because it handles bounds and works well with large sparse Jacobians. dogbox is also bound-aware and can be useful for small problems. lm is Levenberg-Marquardt from MINPACK; it is often fast for small unconstrained problems, but it does not handle bounds.

Method Useful when Watch out for
trf Default robust choice; supports bounds and sparse Jacobians. May spend work managing feasibility near active bounds.
dogbox Small bounded problems with rectangular trust regions. Can be slow if the Jacobian is rank deficient.
lm Small unconstrained problems where Levenberg-Marquardt is a good fit. No bounds; supports only the ordinary linear loss.

Robust Losses Reduce Outlier Leverage

Ordinary least squares treats every squared residual at full strength. That is exactly right when the noise is well-behaved, but a few bad measurements can bend the fit toward themselves.

Robust losses keep the residual-vector interface but replace the simple square with a gentler penalty for large residuals:

\[ \begin{aligned} F(x) &= \frac{1}{2}\sum_{i=1}^{m} \rho\!\left(r_i(x)^2\right),\\ \rho_{\mathrm{soft\_l1}}(z) &= 2\left(\sqrt{1+z}-1\right). \end{aligned} \]

The scalar \(z\) is a squared residual value. Near zero, soft_l1 behaves much like the ordinary square. Far from zero, it grows more slowly, so an outlier still counts but gets less leverage over the fitted curve.

Use Least Squares In SciPy

In SciPy, write a function that returns the residual vector. If you can also write the Jacobian, pass it with jac=...; this saves finite-difference work and makes the local geometry explicit. 

import numpy as np
from scipy import optimize as opt

u = np.array([
    4.0,
    2.0,
    1.0,
    5.0e-1,
    2.5e-1,
    1.67e-1,
    1.25e-1,
    1.0e-1,
    8.33e-2,
    7.14e-2,
    6.25e-2,
])

y = np.array([
    1.957e-1,
    1.947e-1,
    1.735e-1,
    1.6e-1,
    8.44e-2,
    6.27e-2,
    4.56e-2,
    3.42e-2,
    3.23e-2,
    2.35e-2,
    2.46e-2,
])

def model(x, u):
    den = u**2 + x[2] * u + x[3]
    num = u**2 + x[1] * u
    return x[0] * num / den

def residual(x, u, y):
    return model(x, u) - y

def jacobian(x, u, y):
    jac = np.empty((u.size, 4))
    den = u**2 + x[2] * u + x[3]
    num = u**2 + x[1] * u
    scale = -x[0] * num / den**2
    jac[:, 0] = num / den
    jac[:, 1] = x[0] * u / den
    jac[:, 2] = scale * u
    jac[:, 3] = scale
    return jac

x0 = np.array([
    2.5,
    3.9,
    4.15,
    3.9,
])

result = opt.least_squares(
    residual,
    x0=x0,
    jac=jacobian,
    bounds=(0.0, 100.0),
    args=(u, y),
    method="trf",
)

print(result.x)
print(result.cost)
print(result.optimality)

Read the result in least-squares language. result.x is the fitted parameter vector. result.fun is the final residual vector. result.cost is the scalar value \(\frac{1}{2}\|r(x)\|_2^2\) after any selected loss handling. result.optimality reports the first-order optimality measure used by the chosen method.

When To Use It

Use least_squares when Prefer minimize when
Your problem is naturally a vector of residuals from fitting, calibration, or inverse modeling. The objective is a single scalar expression without meaningful residual structure.
You can provide a dense, sparse, or structured residual Jacobian. You need general nonlinear constraints beyond simple parameter bounds.
You need bounds, robust losses, or residual diagnostics from the fitted model. The loss is custom enough that building the scalar objective directly is clearer.
The residual units and weights have a meaningful interpretation. Different terms in the objective are not comparable as residual errors.
Practical habit: scale residuals before fitting. Least squares treats residual magnitude as meaning, so units and weights can decide the solution as much as the optimizer does.
