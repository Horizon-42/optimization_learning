Krylov trust-region minimization

Trust-Region Truncated Generalized Lanczos / Conjugate Gradient algorithm

trust-krylov solves the same trust-region Newton subproblem as trust-ncg, but solves it more accurately inside a Krylov subspace. It is especially useful when Hessian-vector products are cheap and curvature is difficult or indefinite.

Overview Nelder-Mead BFGS Newton-CG trust-ncg trust-krylov trust-exact

Core Idea

trust-krylov uses the same local quadratic model and trust radius:

\[ \min_{p} m_k(p) = f(x_k)+g_k^\mathsf{T}p+\frac{1}{2}p^\mathsf{T}H_kp \quad \text{subject to} \quad \|p\|\leq\Delta_k. \]

The difference is how the subproblem is solved. Instead of using a truncated CG step alone, trust-krylov solves the subproblem in a growing Krylov subspace:

\[ \mathcal{K}_m(H_k,g_k) = \operatorname{span}\{g_k,H_kg_k,H_k^2g_k,\ldots,H_k^{m-1}g_k\}. \]

Krylov Subspace Solve

Inside the subspace, the large Hessian is represented by a much smaller projected matrix. The GLTR idea is to solve the trust-region problem accurately in that low-dimensional space:

\[ p_m=Z_my_m, \qquad \min_{\|y\|\leq\Delta_k} g_m^\mathsf{T}y+\frac{1}{2}y^\mathsf{T}T_my, \qquad T_m=Z_m^\mathsf{T}H_kZ_m. \]

Rosenbrock Example

On Rosenbrock, trust-krylov can use the full Hessian or just Hessian products. The Hessian-vector product interface keeps the method suitable for large-scale problems:

\[ H_kv \quad\text{is enough to build}\quad \mathcal{K}_m(H_k,g_k). \]

Using SciPy

In SciPy, use method="trust-krylov". Provide jac and either hess or hessp. The Hessian-vector product version is often the point of the method. 

import numpy as np
from scipy.optimize import minimize

def rosen(x):
    return np.sum(100.0 * (x[1:] - x[:-1]**2.0)**2.0 + (1 - x[:-1])**2.0)

def rosen_der(x):
    xm = x[1:-1]
    xm_m1 = x[:-2]
    xm_p1 = x[2:]
    der = np.zeros_like(x)
    der[1:-1] = 200 * (xm - xm_m1**2) - 400 * (xm_p1 - xm**2) * xm - 2 * (1 - xm)
    der[0] = -400 * x[0] * (x[1] - x[0]**2) - 2 * (1 - x[0])
    der[-1] = 200 * (x[-1] - x[-2]**2)
    return der

def rosen_hess_p(x, p):
    x = np.asarray(x)
    Hp = np.zeros_like(x)
    Hp[0] = (1200 * x[0]**2 - 400 * x[1] + 2) * p[0] - 400 * x[0] * p[1]
    Hp[1:-1] = (
        -400 * x[:-2] * p[:-2]
        + (202 + 1200 * x[1:-1]**2 - 400 * x[2:]) * p[1:-1]
        - 400 * x[1:-1] * p[2:]
    )
    Hp[-1] = -400 * x[-2] * p[-2] + 200 * p[-1]
    return Hp

x0 = np.array([1.3, 0.7, 0.8, 1.9, 1.2])
result = minimize(
    rosen,
    x0,
    method="trust-krylov",
    jac=rosen_der,
    hessp=rosen_hess_p,
    options={"gtol": 1e-8, "disp": True},
)

print(result.x)

When To Use It

Use trust-krylov when Prefer another method when
You have gradients and Hessian-vector products for a large smooth problem. You only have function values; derivative-free methods fit that information level better.
The Hessian may be indefinite and a more accurate trust-region subproblem solve is worth extra products. Each Hessian-vector product is very expensive; trust-ncg may be cheaper per outer iteration.
You want fewer nonlinear iterations than trust-ncg on hard curvature. The problem is medium-sized with a cheap dense Hessian; trust-exact may be more direct.
Practical habit: compare both outer iterations and Hessian-vector products. trust-krylov can reduce nonlinear iterations, but it may spend more products inside each trust-region solve.

Continue to trust-exact, the nearly exact dense trust-region method.
