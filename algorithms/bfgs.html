Gradient-based local minimization

Broyden-Fletcher-Goldfarb-Shanno algorithm

BFGS is the classic quasi-Newton workhorse. It uses gradients to infer local curvature, then turns that curvature model into smarter search directions without forming the true Hessian.

Overview Nelder-Mead BFGS Newton-CG trust-ncg trust-krylov trust-exact

Core Idea

Newton's method would step with the inverse Hessian:

\[ x_{k+1}=x_k-\alpha_k \nabla^2 f(x_k)^{-1}\nabla f(x_k). \]

BFGS keeps the Newton shape but replaces the true inverse Hessian with an approximation \(H_k\). The search direction is \(p_k=-H_k\nabla f(x_k)\), and a line search chooses the step length \(\alpha_k\).

\[ x_{k+1}=x_k+\alpha_k p_k, \qquad p_k=-H_k g_k, \qquad g_k=\nabla f(x_k). \]

The BFGS Update

After one step, BFGS compares the displacement in parameter space with the displacement in gradient space:

\[ s_k=x_{k+1}-x_k, \qquad y_k=g_{k+1}-g_k, \qquad \rho_k=\frac{1}{y_k^\mathsf{T}s_k}. \]

The inverse-Hessian approximation is then updated by the rank-two formula

\[ H_{k+1} = (I-\rho_k s_k y_k^\mathsf{T})H_k(I-\rho_k y_k s_k^\mathsf{T}) + \rho_k s_k s_k^\mathsf{T}. \]

This update satisfies the secant condition \(H_{k+1}y_k=s_k\). When \(y_k^\mathsf{T}s_k>0\), it also preserves a positive-definite \(H_k\), so the search direction remains downhill.

Rosenbrock Example

For the \(N\)-variable Rosenbrock function

\[ f(x)=\sum_{i=0}^{N-2}100(x_{i+1}-x_i^2)^2+(1-x_i)^2, \]

the interior gradient terms are

\[ \frac{\partial f}{\partial x_j} = 200(x_j-x_{j-1}^2) -400x_j(x_{j+1}-x_j^2) -2(1-x_j). \]

The endpoint derivatives are

\[ \begin{aligned} \frac{\partial f}{\partial x_0} &= -400x_0(x_1-x_0^2)-2(1-x_0),\\ \frac{\partial f}{\partial x_{N-1}} &= 200(x_{N-1}-x_{N-2}^2). \end{aligned} \]

Using SciPy

In SciPy, use method="BFGS" and pass the gradient through jac. If jac is omitted, SciPy estimates the gradient by finite differences. 

import numpy as np
from scipy.optimize import minimize

def rosen(x):
    return np.sum(100.0 * (x[1:] - x[:-1]**2.0)**2.0 + (1 - x[:-1])**2.0)

def rosen_der(x):
    xm = x[1:-1]
    xm_m1 = x[:-2]
    xm_p1 = x[2:]
    der = np.zeros_like(x)
    der[1:-1] = 200 * (xm - xm_m1**2) - 400 * (xm_p1 - xm**2) * xm - 2 * (1 - xm)
    der[0] = -400 * x[0] * (x[1] - x[0]**2) - 2 * (1 - x[0])
    der[-1] = 200 * (x[-1] - x[-2]**2)
    return der

x0 = np.array([1.3, 0.7, 0.8, 1.9, 1.2])
result = minimize(
    rosen,
    x0,
    method="BFGS",
    jac=rosen_der,
    options={"disp": True},
)

print(result.x)

When To Use It

Use BFGS when Prefer another method when
The objective is smooth and a gradient is available or cheap to estimate. The number of variables is very large; storing a dense \(H_k\) can become expensive.
You want a strong default local optimizer without writing Hessian code. The objective is noisy or nonsmooth enough that gradient differences become misleading.
The problem is unconstrained and moderate-sized. You can provide Hessian-vector products for a large problem; Newton-CG or trust-krylov may scale better.
Practical habit: check the gradient implementation before trusting the optimizer. A sign error in \(g_k\) turns BFGS from a fast method into a very confident wrong turn.

Continue to Newton-CG, which uses Hessian information through conjugate-gradient solves instead of a dense inverse-Hessian estimate.
