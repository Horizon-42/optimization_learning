Nearly exact trust-region minimization

Trust-Region Nearly Exact algorithm

trust-exact solves each trust-region Newton subproblem much more accurately than the iterative large-scale methods. It is the dense, medium-size choice when gradients and full Hessians are available.

Overview Nelder-Mead BFGS Newton-CG trust-ncg trust-krylov trust-exact

Core Idea

trust-exact uses the same trust-region model as trust-ncg and trust-krylov:

\[ \min_{p} m_k(p) = f(x_k)+g_k^\mathsf{T}p+\frac{1}{2}p^\mathsf{T}H_kp \quad \text{subject to} \quad \|p\|\leq\Delta_k. \]

The difference is ambition inside the subproblem. Instead of stopping after a truncated CG or Krylov approximation, trust-exact solves a nonlinear equation so the trust-region step is nearly the exact minimizer of the quadratic model.

Nearly Exact Subproblem Solve

A trust-region minimizer satisfies shifted Newton equations. For a suitable Lagrange multiplier \(\lambda_k\), the step obeys

\[ (H_k+\lambda_k I)p_k=-g_k, \qquad H_k+\lambda_k I \succeq 0, \qquad \lambda_k \geq 0. \]

The multiplier also satisfies complementarity:

\[ \|p_k\|\leq\Delta_k, \qquad \lambda_k(\|p_k\|-\Delta_k)=0. \]

If the unconstrained Newton step is inside the trust radius, then \(\lambda_k=0\). Otherwise, the method searches for a positive \(\lambda_k\) that places \(p_k\) on the boundary.

Dense Hessian Tradeoff

trust-exact needs the explicit Hessian. That rules out the Hessian-vector product shortcut used by Newton-CG, trust-ncg, and trust-krylov, but it can reduce outer iterations when dense factorizations are affordable.

\[ \text{storage}(H_k)=O(N^2), \qquad \text{dense factorization}=O(N^3). \]

Rosenbrock Example

The SciPy tutorial demonstrates trust-exact on the Rosenbrock function with a gradient and full Hessian. In two dimensions the Hessian is

\[ H(x)= \begin{bmatrix} 1200x_0^2-400x_1+2 & -400x_0\\ -400x_0 & 200 \end{bmatrix}. \]

Using SciPy

In SciPy, use method="trust-exact". Provide jac and hess; this method does not support a Hessian-vector product-only interface. 

import numpy as np
from scipy.optimize import minimize

def rosen(x):
    return np.sum(100.0 * (x[1:] - x[:-1]**2.0)**2.0 + (1 - x[:-1])**2.0)

def rosen_der(x):
    xm = x[1:-1]
    xm_m1 = x[:-2]
    xm_p1 = x[2:]
    der = np.zeros_like(x)
    der[1:-1] = 200 * (xm - xm_m1**2) - 400 * (xm_p1 - xm**2) * xm - 2 * (1 - xm)
    der[0] = -400 * x[0] * (x[1] - x[0]**2) - 2 * (1 - x[0])
    der[-1] = 200 * (x[-1] - x[-2]**2)
    return der

def rosen_hess(x):
    x = np.asarray(x)
    H = np.diag(-400 * x[:-1], 1) - np.diag(400 * x[:-1], -1)
    diagonal = np.zeros_like(x)
    diagonal[0] = 1200 * x[0]**2 - 400 * x[1] + 2
    diagonal[-1] = 200
    diagonal[1:-1] = 202 + 1200 * x[1:-1]**2 - 400 * x[2:]
    return H + np.diag(diagonal)

x0 = np.array([1.3, 0.7, 0.8, 1.9, 1.2])
result = minimize(
    rosen,
    x0,
    method="trust-exact",
    jac=rosen_der,
    hess=rosen_hess,
    options={"gtol": 1e-8, "disp": True},
)

print(result.x)

When To Use It

Use trust-exact when Prefer another method when
The problem is smooth, unconstrained, and medium-sized. The problem has thousands of variables; matrix-free Newton-CG or trust-krylov is usually a better fit.
You can compute and store a reliable full Hessian. You only have function values or gradients; Nelder-Mead or BFGS matches that information level better.
Reducing objective and gradient evaluations matters more than avoiding dense factorizations. The Hessian is sparse and large enough that Hessian-vector products are much cheaper than factorization.
Practical habit: reach for trust-exact after you know the full Hessian is trustworthy and moderate in size. It is often elegant, but it asks the linear algebra stack to do real work.
