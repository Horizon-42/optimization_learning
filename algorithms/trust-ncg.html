Trust-region second-order minimization

Trust-Region Newton-Conjugate-Gradient algorithm

trust-ncg keeps Newton-CG's Hessian-aware search, but replaces a line search with a trust radius. Each step must stay inside a region where the local quadratic model is expected to be reliable.

Overview Nelder-Mead BFGS Newton-CG trust-ncg trust-krylov trust-exact

Core Idea

Newton-CG is a line-search method: it finds a direction from a quadratic model, then searches along that direction. trust-ncg instead limits the step first and solves a constrained quadratic subproblem:

\[ \min_{p} m_k(p) = f(x_k)+g_k^\mathsf{T}p+\frac{1}{2}p^\mathsf{T}H_kp \quad \text{subject to} \quad \|p\|\leq\Delta_k. \]

The update is then \(x_{k+1}=x_k+p_k\) if the step gives enough real improvement. The radius \(\Delta_k\) is the algorithm's caution knob: small when the model is unreliable, larger when the model predicts the real objective well.

Trust Radius And Agreement

A trust-region method compares actual reduction against predicted model reduction:

\[ \rho_k = \frac{f(x_k)-f(x_k+p_k)} {m_k(0)-m_k(p_k)}. \]

If \(\rho_k\) is small or negative, the model over-promised and the radius should shrink. If \(\rho_k\) is close to one and the step touched the boundary, the local model is trusted farther next time.

Truncated CG Subproblem Solve

trust-ncg uses conjugate gradients to solve the trust-region subproblem approximately. The inner solve stops when it has found a good enough step, when it hits the trust-region boundary, or when it detects curvature that makes a boundary step preferable.

\[ H_kp \approx -g_k, \qquad \|p\|\leq\Delta_k, \qquad H_kv \text{ can be supplied as a product.} \]

This is why trust-ncg is still suitable for large smooth problems: like Newton-CG, it can use Hessian-vector products instead of dense Hessian factorizations.

Rosenbrock Example

On Rosenbrock, trust-ncg uses the same gradient and Hessian formulas introduced for Newton-CG, but step acceptance is governed by the trust-region model rather than a line search.

Using SciPy

In SciPy, use method="trust-ncg". Provide jac and either a full Hessian through hess or a Hessian-vector product through hessp. 

import numpy as np
from scipy.optimize import minimize

def rosen(x):
    return np.sum(100.0 * (x[1:] - x[:-1]**2.0)**2.0 + (1 - x[:-1])**2.0)

def rosen_der(x):
    xm = x[1:-1]
    xm_m1 = x[:-2]
    xm_p1 = x[2:]
    der = np.zeros_like(x)
    der[1:-1] = 200 * (xm - xm_m1**2) - 400 * (xm_p1 - xm**2) * xm - 2 * (1 - xm)
    der[0] = -400 * x[0] * (x[1] - x[0]**2) - 2 * (1 - x[0])
    der[-1] = 200 * (x[-1] - x[-2]**2)
    return der

def rosen_hess(x):
    x = np.asarray(x)
    H = np.diag(-400 * x[:-1], 1) - np.diag(400 * x[:-1], -1)
    diagonal = np.zeros_like(x)
    diagonal[0] = 1200 * x[0]**2 - 400 * x[1] + 2
    diagonal[-1] = 200
    diagonal[1:-1] = 202 + 1200 * x[1:-1]**2 - 400 * x[2:]
    return H + np.diag(diagonal)

x0 = np.array([1.3, 0.7, 0.8, 1.9, 1.2])
result = minimize(
    rosen,
    x0,
    method="trust-ncg",
    jac=rosen_der,
    hess=rosen_hess,
    options={"gtol": 1e-8, "disp": True},
)

print(result.x)

When To Use It

Use trust-ncg when Prefer another method when
You have gradients and Hessian or Hessian-vector products for a smooth unconstrained problem. You only have function values; derivative-free methods are a better starting point.
Newton-CG line-search steps are unstable because the Hessian is ill-conditioned. You want a more accurate trust-region subproblem solve; trust-krylov is the next candidate.
The problem is large and second-order information can be applied as products. The problem is medium-sized with a cheap dense Hessian; trust-exact can take fewer outer iterations.
Practical habit: watch both the gradient norm and the trust radius. A shrinking radius means the local model is being treated with suspicion, even if the computed Newton-like direction looks decisive.

Continue to trust-krylov, which solves the trust-region subproblem more accurately in a Krylov subspace.
